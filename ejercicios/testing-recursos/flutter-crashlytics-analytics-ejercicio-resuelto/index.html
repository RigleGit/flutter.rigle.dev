Firebase Crashlytics y Analytics en Flutter: ejercicio resuelto

Crashlytics registra automáticamente los crashes de tu app y los agrupa por causa. Analytics recoge eventos de comportamiento del usuario. Juntos son la base de la observabilidad de cualquier app en producción.

Enunciado

Integra Firebase Crashlytics y Analytics en una app Flutter que:

  • Registra crashes y errores no fatales con contexto (keys, logs).
  • Lanza un crash de prueba desde la UI.
  • Registra eventos de Analytics personalizados.
  • Registra la pantalla actual automáticamente al navegar.
  • Identifica al usuario autenticado en Crashlytics.

Configuración de AndroidManifest y Info.plist

Crashlytics no requiere configuración adicional en el manifiesto. Asegúrate de que el archivo google-services.json (Android) y GoogleService-Info.plist (iOS) estén en las rutas correctas.

import 'dart:async';
import 'package:firebase_analytics/firebase_analytics.dart';
import 'package:firebase_core/firebase_core.dart';
import 'package:firebase_crashlytics/firebase_crashlytics.dart';
import 'package:flutter/foundation.dart';
import 'package:flutter/material.dart';

// ── Servicio de observabilidad ─────────────────────────────────────────────────
class ObservabilityService {
  static final _crashlytics = FirebaseCrashlytics.instance;
  static final _analytics = FirebaseAnalytics.instance;

  // Inicializar y configurar handlers globales de errores
  static Future<void> initialize() async {
    // Pasar todos los errores de Flutter a Crashlytics
    FlutterError.onError = _crashlytics.recordFlutterFatalError;

    // Capturar errores async fuera del árbol de widgets
    PlatformDispatcher.instance.onError = (error, stack) {
      _crashlytics.recordError(error, stack, fatal: true);
      return true;
    };

    // En debug, deshabilitar Crashlytics (solo activo en release)
    await _crashlytics.setCrashlyticsCollectionEnabled(!kDebugMode);
  }

  // ── Crashlytics ──────────────────────────────────────────────────────────────

  /// Identifica al usuario para que los crashes aparezcan con su ID
  static Future<void> setUser({required String userId, String? email}) async {
    await _crashlytics.setUserIdentifier(userId);
    await _crashlytics.setCustomKey('user_email', email ?? 'unknown');
  }

  /// Añade contexto antes de una operación crítica
  static Future<void> setContext(String key, String value) =>
      _crashlytics.setCustomKey(key, value);

  /// Añade un log visible en el informe de crash
  static void log(String message) => _crashlytics.log(message);

  /// Registra un error no fatal (no mata la app, pero sí aparece en Crashlytics)
  static Future<void> recordError(
    Object error,
    StackTrace? stack, {
    String? reason,
    bool fatal = false,
  }) async {
    await _crashlytics.recordError(
      error,
      stack,
      reason: reason,
      fatal: fatal,
    );
  }

  // ── Analytics ────────────────────────────────────────────────────────────────

  /// Registra la pantalla actual (llamar al montar cada ruta)
  static Future<void> setCurrentScreen(String screenName) =>
      _analytics.setCurrentScreen(screenName: screenName);

  /// Evento de producto visto
  static Future<void> logProductView({
    required String productId,
    required String productName,
    required double price,
  }) =>
      _analytics.logViewItem(
        items: [
          AnalyticsEventItem(
            itemId: productId,
            itemName: productName,
            price: price,
          )
        ],
      );

  /// Evento de compra completada
  static Future<void> logPurchase({
    required String transactionId,
    required double value,
    required String currency,
  }) =>
      _analytics.logPurchase(
        transactionId: transactionId,
        value: value,
        currency: currency,
      );

  /// Evento personalizado
  static Future<void> logEvent(
    String name, {
    Map<String, Object>? parameters,
  }) =>
      _analytics.logEvent(name: name, parameters: parameters);

  /// Propiedad de usuario (segmentación en Analytics)
  static Future<void> setUserProperty({
    required String name,
    required String value,
  }) =>
      _analytics.setUserProperty(name: name, value: value);
}

// ── Main ───────────────────────────────────────────────────────────────────────
Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();

  // Capturar errores antes de Firebase.initializeApp
  runZonedGuarded(
    () async {
      await Firebase.initializeApp();
      await ObservabilityService.initialize();

      // Simular usuario autenticado
      await ObservabilityService.setUser(
        userId: 'user_12345',
        email: 'demo@flutter.dev',
      );

      runApp(const CrashlyticsDemo());
    },
    (error, stack) {
      FirebaseCrashlytics.instance.recordError(error, stack, fatal: true);
    },
  );
}

// ── App ────────────────────────────────────────────────────────────────────────
class CrashlyticsDemo extends StatelessWidget {
  const CrashlyticsDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Crashlytics + Analytics',
      navigatorObservers: [
        // Registrar automáticamente la pantalla en Analytics al navegar
        FirebaseAnalyticsObserver(analytics: FirebaseAnalytics.instance),
      ],
      home: const HomePage(),
    );
  }
}

// ── Pantalla principal ─────────────────────────────────────────────────────────
class HomePage extends StatefulWidget {
  const HomePage({super.key});
  @override
  State<HomePage> createState() => _HomePageState();
}

class _HomePageState extends State<HomePage> {
  final _logs = <String>[];

  @override
  void initState() {
    super.initState();
    ObservabilityService.setCurrentScreen('home');
    ObservabilityService.log('HomePage montada');
  }

  void _addLog(String msg) =>
      setState(() => _logs.insert(0, '${DateTime.now().toIso8601String().substring(11, 19)} $msg'));

  Future<void> _testCrash() async {
    ObservabilityService.log('Usuario pulsa crash de prueba');
    await ObservabilityService.setContext('last_action', 'test_crash');
    // En release esto enviaría el crash a Crashlytics y cerraría la app
    FirebaseCrashlytics.instance.crash();
    _addLog('Crash de prueba enviado');
  }

  Future<void> _logNonFatal() async {
    try {
      throw StateError('Estado inválido al procesar el pedido #42');
    } catch (e, stack) {
      await ObservabilityService.recordError(
        e,
        stack,
        reason: 'Error al procesar pedido en checkout',
        fatal: false,
      );
      _addLog('Error no fatal registrado en Crashlytics');
    }
  }

  Future<void> _logProductView() async {
    await ObservabilityService.logProductView(
      productId: 'book_flutter_001',
      productName: 'Flutter en Profundidad',
      price: 29.99,
    );
    _addLog('Analytics: view_item registrado');
  }

  Future<void> _logPurchase() async {
    await ObservabilityService.logPurchase(
      transactionId: 'txn_${DateTime.now().millisecondsSinceEpoch}',
      value: 29.99,
      currency: 'EUR',
    );
    _addLog('Analytics: purchase registrado');
  }

  Future<void> _logCustomEvent() async {
    await ObservabilityService.logEvent(
      'tutorial_complete',
      parameters: {
        'tutorial_id': 'flutter_basics',
        'success': true,
      },
    );
    _addLog('Analytics: tutorial_complete registrado');
  }

  Future<void> _setUserProperty() async {
    await ObservabilityService.setUserProperty(
      name: 'subscription_plan',
      value: 'premium',
    );
    _addLog('Analytics: user property subscription_plan=premium');
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Crashlytics + Analytics')),
      body: Padding(
        padding: const EdgeInsets.all(16),
        child: Column(
          crossAxisAlignment: CrossAxisAlignment.stretch,
          children: [
            // ── Crashlytics ────────────────────────────────────────────
            Text('Crashlytics',
                style: Theme.of(context).textTheme.titleMedium
                    ?.copyWith(color: Theme.of(context).colorScheme.primary)),
            const SizedBox(height: 8),
            FilledButton.icon(
              onPressed: _logNonFatal,
              icon: const Icon(Icons.warning_amber),
              label: const Text('Registrar error no fatal'),
              style: FilledButton.styleFrom(backgroundColor: Colors.orange),
            ),
            const SizedBox(height: 4),
            OutlinedButton.icon(
              onPressed: _testCrash,
              icon: const Icon(Icons.dangerous),
              label: const Text('Crash de prueba (solo release)'),
              style: OutlinedButton.styleFrom(foregroundColor: Colors.red),
            ),

            const Divider(height: 24),

            // ── Analytics ──────────────────────────────────────────────
            Text('Analytics',
                style: Theme.of(context).textTheme.titleMedium
                    ?.copyWith(color: Theme.of(context).colorScheme.primary)),
            const SizedBox(height: 8),
            Wrap(
              spacing: 8,
              runSpacing: 8,
              children: [
                FilledButton.tonal(
                  onPressed: _logProductView,
                  child: const Text('view_item'),
                ),
                FilledButton.tonal(
                  onPressed: _logPurchase,
                  child: const Text('purchase'),
                ),
                FilledButton.tonal(
                  onPressed: _logCustomEvent,
                  child: const Text('tutorial_complete'),
                ),
                FilledButton.tonal(
                  onPressed: _setUserProperty,
                  child: const Text('user property'),
                ),
              ],
            ),

            const Divider(height: 24),

            Text('Registro local:',
                style: Theme.of(context).textTheme.labelLarge),
            const SizedBox(height: 8),
            Expanded(
              child: _logs.isEmpty
                  ? const Center(
                      child: Text('Pulsa un botón para registrar un evento',
                          style: TextStyle(color: Colors.grey)))
                  : ListView.builder(
                      itemCount: _logs.length,
                      itemBuilder: (_, i) => Text(_logs[i],
                          style: const TextStyle(fontSize: 12,
                              fontFamily: 'monospace')),
                    ),
            ),
          ],
        ),
      ),
    );
  }
}

Datos que Crashlytics captura automáticamente

DatoFuente
Stack traceFlutterError.onError + PlatformDispatcher.onError
Versión de apppubspec.yaml version
Dispositivo y OSSDK de Firebase
Custom keyssetCustomKey
Logslog() (últimos 64 KB)
UsuariosetUserIdentifier

Errores frecuentes

  • Crashes no aparecen en la consola Firebase: Crashlytics solo envía reportes en modo release. En debug, usa kDebugMode para evitar colisiones. Espera también hasta 5 minutos para que aparezcan.
  • Olvidar runZonedGuarded: los errores async que ocurren fuera del árbol de widgets (timers, isolates) no son capturados por FlutterError.onError. runZonedGuarded los captura todos.
  • Enviar datos PII a Analytics: los eventos de Analytics no deben incluir nombres, emails ni IDs en claro. Usa IDs anonimizados y configura setAnalyticsCollectionEnabled(false) cuando el usuario rechace el consentimiento.

FAQ

¿Puedo usar Crashlytics sin Analytics?

Sí. Son paquetes independientes aunque complementarios. Puedes instalar solo firebase_crashlytics si no necesitas Analytics.

¿Cómo pruebo que Crashlytics funciona sin publicar la app?

Usa FirebaseCrashlytics.instance.crash() en un build de release local (flutter build apk --release). Abre la app, pulsa el botón y en pocos minutos verás el crash en la consola Firebase.

¿Puedo cumplir el RGPD con Firebase Analytics?

Sí. Desactiva la colección con setAnalyticsCollectionEnabled(false) hasta que el usuario acepte el consentimiento. Usa resetAnalyticsData() si el usuario revoca el consentimiento.
